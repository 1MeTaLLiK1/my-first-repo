# Шпаргалка по основным командам Git

## 1. git init
**Назначение:** Инициализирует новый репозиторий Git в текущей папке.
**Когда использовать:** Когда начинаете новый проект с нуля.
**Пример в терминале:**
git init

## 2. git clone
**Назначение:** Копирует удаленный репозиторий с GitHub на ваш компьютер.
**Когда использовать:** Когда хотите работать с существующим проектом с GitHub.
**Пример:**
git clone https://github.com/user/project.git

## 3. git add
**Назначение:** Добавляет файлы в "индекс" (staging area) для следующего коммита.
**Когда использовать:** После внесения изменений в файлы.
**Примеры:**
git add file.txt # добавить конкретный файл
git add . # добавить все измененные файлы

## 4. git commit
**Назначение:** Фиксирует изменения с комментарием.
**Когда использовать:** После `git add`, чтобы сохранить "снимок" текущего состояния.
**Пример:**
git commit -m "Добавил навигационное меню"

## 5. git push
**Назначение:** Отправляет ваши коммиты на удаленный репозиторий (GitHub).
**Когда использовать:** Когда хотите поделиться изменениями или сохранить их на GitHub.
**Пример:**
git push origin main

## 6. git pull
**Назначение:** Загружает изменения с GitHub и объединяет их с вашей локальной версией.
**Когда использовать:** Перед началом работы, чтобы получить последние обновления от других.
**Пример:**
git pull origin main
